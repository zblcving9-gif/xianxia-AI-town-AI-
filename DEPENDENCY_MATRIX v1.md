## 运行时依赖说明

虽然文件之间没有直接的引用关系（符合正交分解原则），但各模块在运行时通过**全局对象**进行协作：

### 全局对象通信机制

| 全局对象 | 定义位置 | 被使用位置 |
|----------|----------|------------|
| `Game` | index.html | 所有系统 |
| `GameActions` | index.html | 所有系统 |
| `Core` | core.js | state.js, combat.js, building.js 等 |
| `Renderer` | renderer.js | index.html |
| `GameState` | state.js | 所有系统 |
| `GameData` | state.js | 所有需要数据定义的系统 |
| `Physics` | core.js | combat.js 等 |
| `Graphics` | renderer.js | renderer.js 内部 |
| `BuildingSystem` | building.js | survival.js, crafting.js 等 |
| `ResourceSystem` | resource.js | index.html |
| `CombatSystem` | combat.js | cultivation.js, dialog.js 等 |
| `FactionSystem` | faction.js | dialog.js, cultivation.js, social.js 等 |
| `WeatherSystem` | weather.js | dialog.js, cultivation.js, survival.js 等 |
| `DialogSystem` | dialog.js | index.html |
| `CultivationSystem` | cultivation.js | index.html |
| `CraftingSystem` | crafting.js | index.html |
| `SocialSystem` | social.js | state.js |
| `SurvivalSystem` | survival.js | index.html |
| `SpiritualSystem` | spiritual.js | cultivation.js, combat.js 等 |

## 如何运行

1. 将项目文件放置在Web服务器目录下
2. 用浏览器访问 `index.html`
3. 推荐使用本地服务器（如 `python -m http.server` 或 VS Code Live Server）

```bash
# 启动本地服务器示例
cd /workspace
python3 -m http.server 8080
# 然后访问 http://localhost:8080
```
